01 — Overview

What is FieldWatt?

FieldWatt is a two-sided field operations platform for electricity meter reading companies (infracon businesses). Admins upload area/property data via Excel, assign work to agents, and monitor all activity from a web dashboard. Agents use a mobile app to see their assigned meters, visit locations, record readings, upload photos, and mark visit outcomes — all with full offline support.

The system closes the loop between field data collection and back-office management — eliminating paper forms, missed visits, and opaque agent performance.

🏢

Admin web dashboard

Upload data, assign work, track progress, export reports. Runs in any browser.

📱

Agent mobile app

React Native — one codebase for Android and iOS. Works fully offline.

⚡

Real-time sync

Agent updates flow to admin dashboard live when online, queue when offline.

📊

Analytics & export

Completion rates, agent performance, area coverage — downloadable anytime.

02 — Personas

Who uses this?

⚙️ Admin (you)
  • Uploads property data via Excel
  • Assigns areas or serial ranges to agents
  • Monitors daily login & progress
  • Handles leave — reassigns work
  • Exports completion reports
  • Reviews flagged or problem visits
  • Manages agent accounts
🧑‍🔧 Field Agent
  • Logs in, sees today's assigned list
  • Navigates to each property
  • Enters meter reading value
  • Photos meter or situation
  • Marks outcome (done / locked / refused…)
  • Works fully offline if needed
  • Marks self on leave
03 — Admin panel

Admin features

Must have Excel upload & property ingestion
  • Upload .xlsx file; system parses and structures by Area → Property type (Flat / Bungalow / Raw house)
  • Preview parsed data before confirming import
  • Validate required columns — show errors row by row if data is bad
  • Each property gets a serial number, consumer name, address, meter number
  • Support re-upload to update existing data (merge mode, not wipe)
Must have Area-wise property browser
  • View all areas (Bavdhan, Chinchwad, etc.) with flat/bungalow/raw house counts
  • Drill into area → see all properties with serial numbers
  • Filter by property type, status (pending / done / problem)
  • Search by consumer name or meter number
Must have Agent assignment engine
  • Assign by area (entire area to one or more agents)
  • Assign by serial number range (e.g. 1–100 to Agent X)
  • Assign individual properties by selection
  • Multiple agents can share one area
  • See assignment coverage — what's assigned, what's still unassigned
  • Partial completion reassignment — if agent did 300/500, auto-select remaining 200 and reassign
Must have Live agent dashboard
  • Today's attendance: who has logged in vs not yet
  • Per-agent completion counter: Assigned / Done / Pending / Problem
  • Last active timestamp per agent
  • Mark agent on leave → system prompts to reassign their pending work
  • Drill down into any agent's list and see each property's status + photo
Must have Data export
  • Export full area data as Excel with reading values, status, agent, timestamp
  • Filter export: by area, date range, agent, status
  • Export agent performance summary
  • Auto-generate daily report (CSV or PDF)
New — Added Live map view Research finding
  • See all agents as dots on a map (GPS location when online)
  • See property pins — green = done, red = pending, orange = problem
  • Helps admin spot if agent is stuck, going wrong area, or skipping locations
  • Heat map of pending work density per area
New — Added Anomaly & tamper alerts Research finding
  • Flag readings that are 0, abnormally low, or 50%+ different from previous month
  • Admin gets alert list: "12 readings look unusual today"
  • Mark flagged reading as verified or escalate for physical inspection
  • Possible meter tampering flag when reading drops significantly
Should have Notification & broadcast
  • Push notification to all agents (e.g. "Today's target: 100 readings by 4 PM")
  • Push to specific agent
  • In-app chat message to an agent
04 — Agent mobile app

Agent app features

Must have Today's work list
  • On login, agent sees all assigned properties sorted by serial number
  • Each item shows: Serial no. · Consumer name · Address · Property type · Status
  • Progress bar: "48 of 100 done"
  • Pending / done / problem tabs for easy filtering
  • Tap any item to open the reading form
Must have Reading submission form
  • Enter meter reading number (numeric keypad)
  • Select visit outcome (see status codes section)
  • Upload photo — required for "Done" and "Door locked" outcomes
  • Add text note (optional, free text)
  • GPS auto-captured on submit
  • Timestamp auto-captured on submit
  • If offline: save to local queue, auto-sync when signal returns
Must have Offline-first operation
  • All assigned data downloaded to device on login
  • Agent can complete full day of readings with zero internet
  • Photos stored locally until sync
  • Sync queue shows: "14 records waiting to upload"
  • Auto-syncs in background when signal detected
  • Conflict resolution: server wins for assignments, local wins for readings
New — Added Navigation assist Research finding
  • "Get directions" button → opens Google Maps / Apple Maps to property address
  • Route suggestion: order today's list by geographic proximity to reduce walking
  • Area map showing all pending properties as pins
New — Added Reading validation Research finding
  • If reading entered is lower than last month — show warning before allowing submit
  • If reading is suspiciously high (10x normal) — confirm prompt
  • Prevents typo errors at source, not discovered later in billing
Should have Leave & attendance
  • Agent can mark "I am on leave today" from the app
  • Admin gets notification and can reassign the work
  • Attendance log visible to admin (login time, last activity time)
Nice to have Performance summary for agent
  • Agent can see own stats: this week / this month
  • Total done, problems encountered, average per day
  • Encourages self-motivation and transparency
05 — Visit status codes

What an agent can mark

Every visit gets one of these outcomes. Each has photo requirement rules and flows to the right admin view.

Reading taken ✅

Reading value entered + meter photo uploaded

Door locked 🔒

Photo of locked door required. Admin notes for revisit

Not reachable 📵

Attempted call, no answer. Timestamp auto-logged

Access denied ❌

Customer present but refused. Note required

Meter not found 🔍

Cannot locate meter at address. Note + photo

Meter damaged ⚠️

Visible damage or suspected tamper. Photo mandatory

Revisit needed 🔄

First visit incomplete, schedule follow-up

Vacant property 🏚️

Address is empty / under construction

06 — What we're adding

Beyond the original brief Research-backed additions

After researching industry-standard field ops platforms like Fieldmaster, Axonator, TrackoField, DCS, and Itron's Temetra, here are the gaps we're filling:

Things research showed that the original plan was missing:

New GPS geo-verified check-in
  • Reading can only be submitted when agent is within ~100m of property GPS coordinates
  • Prevents fake "done" submissions from home or office
  • GPS coordinates stored with every reading — admin can verify location on map
  • Override available for admin if GPS is unavailable in a specific area
New Previous reading history on agent screen
  • When agent opens a property, show last 2–3 months reading values
  • Helps agent spot if current reading looks wrong before submitting
  • Shows estimated consumption (e.g. avg 120 units/month for this consumer)
New Revisit scheduling
  • Door-locked or not-reachable → admin can schedule a revisit date
  • System re-queues the property in agent's list on that date
  • Revisit counter per property — if 3 failed attempts, escalates to supervisor
New Auto daily report
  • Every evening at set time, auto-generate area completion summary
  • Emailed to admin: readings done today, pending count, problems flagged
  • No manual pulling of reports required for routine oversight
New Route optimization for agents
  • If property GPS coordinates are available, suggest an optimized walking order
  • Reduces total travel time per agent per day
  • Based on geographic clustering — nearest unvisited property next
New Multi-cycle / monthly data separation
  • Each upload cycle gets a "cycle" label (e.g. June 2025)
  • Historical reading data is preserved per cycle
  • Admin can compare completion rate month over month
  • Useful for detecting consistently unreachable consumers
07 — Tech stack

Technology decisions

Layer Choice Why
Admin frontend React + Vite Fast dev, component reuse, good table/chart ecosystem
Agent mobile app React Native (Expo) Single codebase → Android + iOS. Expo simplifies camera, GPS, offline
Backend API Node.js + Express Easy for solo dev, huge ecosystem, same language as frontend
Primary database PostgreSQL Relational — perfect for area/property/agent/assignment structure
Mobile offline DB SQLite (via expo-sqlite) Native SQLite on device, syncs to Postgres on reconnect
Photo storage Cloudinary Free tier generous, automatic compression, CDN delivery
Authentication JWT + bcrypt Simple, stateless, works offline-friendly
Push notifications Expo Push + Firebase FCM Free, cross-platform, works with Expo out of the box
Maps react-native-maps + Google Maps API Best map integration, directions API for navigation
Excel parsing SheetJS (xlsx) Parse .xlsx server-side, battle-tested, free
Hosting (API) Railway Postgres + Node.js in one place, free tier, easy deploy
Hosting (frontend) Vercel Zero config deploy for React, free tier
Email (reports) Resend / Nodemailer Simple transactional email for daily reports

10 — Risks

Risks & how to handle them

Risk
Severity
Mitigation
Excel files have inconsistent column names from different sources
High
Build a column mapping UI — admin maps their Excel columns to system fields on first upload
Agents in low-connectivity areas can't sync for hours
Med
Full offline-first design with local SQLite. Sync queue survives app close/restart
Agent submits reading without physically going to property
High
GPS geo-verification (100m radius check). Admin can review GPS trail per agent
Photo upload eats agent's mobile data
Med
Compress photos to ~200KB before upload. Agent can set "upload on WiFi only" preference
Duplicate readings if agent submits and app crashes before sync confirm
Med
Idempotency keys on reading submission. Server deduplicates by (property_id + cycle_id)
Cloudinary photo storage costs at scale
Low
Free tier handles ~25GB. Purge photos older than 3 cycles. Switch to S3 if needed
iOS App Store review delay for updates
Med
Use Expo OTA updates for JS-only changes. Store review only needed for native module changes
